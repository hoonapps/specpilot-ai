# SpecPilot AI 제품 기획서

## 1. 서비스 개요

SpecPilot AI는 컴퓨터 견적과 노트북 구매를 위한 AI 의사결정 에이전트다. 사용자가 예산, 사용 목적, 모니터 해상도, 선호 조건, 제외 조건을 입력하면 에이전트가 후보 견적과 노트북을 수집하고, 부품 호환성, 성능 병목, 가격, 리뷰, 벤치마크 근거를 함께 분석해 최종 추천 리포트를 생성한다.

핵심 컨셉은 최저가 검색이 아니라 구매 실패 가능성을 줄이는 것이다. 컴퓨터 구매에서 사용자가 실제로 어려워하는 지점은 CPU와 GPU의 균형, 메인보드 소켓, 파워 용량, 케이스 간섭, RAM/SSD 구성, 노트북 발열과 팬 소음, 옵션별 가격 차이다. SpecPilot AI는 이 판단을 구조화한다.

초기 제품명: SpecPilot AI

대상 MVP:

- 데스크톱 PC 견적
- 노트북 구매 비교

초기 대표 요청:

> 영상 편집과 게임용 데스크톱 200만원 안에서 맞춰줘. QHD 144Hz 모니터를 쓸 거고 업그레이드 여지도 있었으면 좋겠어.

## 2. 문제 정의

- 가격 비교 페이지는 부품별 최저가를 보여주지만 전체 견적의 호환성과 병목을 충분히 설명하지 못한다.
- 초보자는 CPU, GPU, 메인보드, RAM, 파워, 케이스의 조합이 맞는지 판단하기 어렵다.
- 노트북은 같은 모델이라도 RAM, SSD, GPU, 디스플레이 옵션에 따라 가격과 체감 성능이 크게 다르다.
- 리뷰와 커뮤니티 의견은 많지만 목적별로 어떤 불만이 중요한지 정리하기 어렵다.
- 할인, 배송비, 조립비, OS 포함 여부, 카드 혜택 때문에 실구매가 계산이 복잡하다.

## 3. 사용자와 주요 시나리오

| 사용자 | 상황 | 성공 기준 |
|---|---|---|

| 크리에이터 | 영상 편집용 PC 또는 노트북을 고른다 | CPU/GPU/RAM/SSD 병목과 업그레이드 가치를 이해한다 |

| 소규모 사업자 | 사무용 PC와 노트북을 반복 구매한다 | 가격, AS, 재고, 표준 구성을 빠르게 비교한다 |

## 5. LangGraph 에이전트 설계

LangGraph는 각 단계를 노드로 분리하고 상태 객체에 사용자 조건, 후보 제품, 가격 스냅샷, 리뷰 인사이트, 호환성 검증, 점수표, 출처, 최종 리포트를 누적한다.

노드 흐름:

1. Intent Parser - 자연어 요청을 예산, 목적, 카테고리, 필수 조건으로 구조화
2. Clarifier - 모호한 예산, 해상도, 휴대성, 조립 여부를 짧게 확인
3. Query Planner - 가격, 스펙, 벤치마크, 리뷰 검색 계획 생성
4. Product Collector - 데스크톱 견적 또는 노트북 후보 수집
5. Deduplicator - 모델명과 옵션 차이를 정규화하고 중복 제거
6. Compatibility Checker - CPU/보드/RAM/PSU/케이스 호환성 확인
7. Price Tracker - 배송비, 조립비, 쿠폰을 반영한 실구매가 계산
8. Review Analyzer - 반복 불만, 발열, 소음, AS, 리뷰 리스크 요약
9. Scoring Engine - 목적별 가중치로 종합 점수 산출
10. Verifier - 출처 누락, 가격 시점, 호환성 근거 부족 확인
11. Report Writer - 비교표와 최종 추천 생성

상태 모델:

```text
PurchaseState = {
  user_query,
  criteria,
  search_plan,
  candidates,
  normalized_products,
  compatibility_notes,
  price_snapshots,
  review_insights,
  scorecards,
  citations,
  verification_flags,
  final_report
}
```

## 6. 지식 그래프 설계

Neo4j는 단순 저장소가 아니라 구매 판단 근거를 관계로 연결하는 그래프 메모리로 사용한다.

```cypher
(:Build)-[:USES]->(:Component)
(:Build)-[:SOLD_AS]->(:Offer)-[:OFFERED_BY]->(:Seller)
(:Build)-[:CHECKED_BY]->(:CompatibilitySignal)
(:Component)-[:HAS_BENCHMARK]->(:Benchmark)
(:Laptop)-[:HAS_REVIEW]->(:Review)
```

대표 노드:

- Build: 정규화된 데스크톱 견적 조합
- Laptop: 노트북 완제품
- Component: CPU, GPU, RAM, SSD, PSU, Case
- Offer: 판매 가격, 배송비, 조립비, 쿠폰
- Seller: 가격 비교 출처, 오픈마켓, 공식 스토어
- Benchmark: 성능 근거
- Review: 사용자 리뷰와 전문 리뷰
- CompatibilitySignal: 소켓, 파워, 케이스 간섭 같은 검증 신호

| 평가 축 | 가중치 | 설명 |

## 12. 공개 베타 신뢰 정책

공개 베타에서는 추천 결과의 설득력보다 신뢰 장치를 먼저 노출한다. 사용자는 각 추천이 어떤 출처를 근거로 삼았는지, 가격이 얼마나 오래 캐시되는지, 어떤 근거가 관리자 검수 대상인지 확인할 수 있어야 한다.

현재 제품 구현:

- `/intake/diagnose`는 분석 전 구매 조건을 진단해 준비도 점수, 누락 슬롯, 추가 질문, 추천 필수/제외 조건, 정규화된 분석 요청을 반환한다.
- 루트 웹 UI는 조건 진단 버튼으로 추천 조건을 적용하거나 진단된 조건으로 바로 분석을 실행할 수 있다.
- `/policy/trust` API로 캐시 정책, 제휴 고지, 공정성 규칙, 리뷰 표현 규칙을 공개한다.
- `report.source_trust`는 출처별 신뢰 등급, 신뢰도, freshness, cache TTL, 검수 필요 여부를 반환한다.
- `report.trust_policy`는 가격 만료 처리, 제휴 링크 고지, 비제휴 대안 노출 원칙을 반환한다.
- 웹 결과 화면은 출처 신뢰도와 캐시/제휴 정책을 분석 결과 옆에 표시한다.
- 신뢰도 0.8 미만 또는 리스크 플래그가 있는 수집 근거는 관리자 검수 큐로 들어간다.
- `/sources/ingest-url`은 외부 상품 URL 또는 HTML 스냅샷에서 제목, 가격, 본문 근거를 추출해 `SourceCandidate`와 검수 항목을 만든다.
- URL 인입은 내부망, localhost, private IP를 차단하고 실제 추천 반영 전 운영자 검수를 요구한다.
- 관리자 콘솔은 상품 URL과 기대 모델명을 입력해 실제 페이지 근거를 검수 큐로 보낼 수 있다.
- `/sources/monitors`는 반복 확인이 필요한 상품 URL을 워크스페이스별 모니터로 등록한다.
- `/sources/refresh`는 등록된 URL 모니터를 다시 수집하고 성공/실패, live fetch 여부, 연결된 검수 항목을 `source_refresh_runs` 이력으로 남긴다.
- `/sources/schedule`은 `cadence_minutes`와 마지막 실행 시각을 기준으로 due 모니터와 upcoming 모니터를 분리한다.
- `/sources/refresh-due`는 지금 실행해야 할 due 모니터만 수집해 외부 cron 또는 Cloud Scheduler가 호출할 운영 경계를 제공한다.
- 관리자 콘솔은 URL 모니터와 수집 refresh 이력을 보여줘 가격 근거 운영 상태를 추적하게 한다.
- `/sources/providers`는 live fetch 허용 host, robots/약관 검토 상태, credential 상태, 시간당 rate limit을 워크스페이스별 정책으로 저장한다.
- `/sources/providers/check`는 URL이 provider 정책과 rate limit을 통과하는지 확인하고, 정책이 없는 live fetch는 차단한다.
- live fetch 실행 시 provider 정책과 시간당 사용량 로그를 남겨 무분별한 외부 호출을 막는다.
- `/ops/integrations`는 가격 API, 오픈마켓, 공식 스토어, 리뷰 feed, 벤치마크, 이메일, webhook, observability, 제휴, scheduler provider의 credential 상태, rate limit, 보존 기간, 운영 증거를 저장한다.
- `/ops/integration-readiness`는 공개 전 필수 외부 연동을 mock/configured/verified/blocker 상태로 집계하고 미등록 핵심 연동을 출시 게이트 blocker로 올린다.
- `/reports/{report_id}/purchase-links`는 저장 리포트 후보별 제휴/비제휴 구매 링크, 판매처, 가격, 쿠폰, 제휴 네트워크, 노출 순서를 저장한다.
- `/reports/{report_id}/purchase-link-governance`는 제휴 링크 단독 노출 여부, 비제휴 대안 유무, 링크 클릭 수를 집계해 공개 전 보강 액션을 반환한다.
- `/buy/{link_id}`는 공개 리포트의 구매 링크 클릭을 기록한 뒤 실제 판매처로 redirect해 추천 순위와 제휴 수익, 클릭 지표를 분리한다.
- `/alerts/evaluate`는 활성 알림 구독을 평가하고 목표가 도달 시 발송 큐 이벤트를 만든다.
- `/alerts/events`는 워크스페이스별 알림 발송 이벤트를 조회한다.
- `/alerts/channels`는 이메일, 웹훅, SMS 발송 채널을 워크스페이스별로 설정하고 대상은 마스킹해 반환한다.
- `/alerts/dispatch`는 queued/failed 알림 이벤트를 채널별 outbox로 발송 처리하고 성공, 실패, 재시도 예정 상태를 기록한다.
- `/alerts/deliveries`는 워크스페이스별 발송 시도 이력을 조회하게 한다.
- 운영 지표는 알림 구독 수, 발송 이벤트 수, 트리거 수, 채널 수, 발송 성공/실패 수를 함께 노출한다.
- `quality_audit`는 분석별 품질 점수, 예상 소스 호출, 예상 토큰/비용, 경고 수, 공개 차단 사유를 포함한다.
- `/ops/quality`는 워크스페이스별 품질 감사 대시보드를 반환한다.
- `/ops/regression`은 최근/이전 분석 구간의 품질 점수, 분석당 비용, blocker/warning 변화와 provider 차단율을 비교해 release health를 반환한다.
- 관리자 콘솔은 품질/비용 감사 섹션에서 최근 분석의 공개 가능성을 보여준다.
- 관리자 콘솔은 품질 회귀 모니터에서 품질 하락, 비용 급등, provider 차단율 리스크와 다음 액션을 보여준다.
- 분석 실행 시 LangGraph 단계별 이벤트는 `trace_spans` 테이블에 별도 저장된다.
- `/ops/traces`는 워크스페이스별 trace 요약, 품질 점수, span 수를 반환한다.
- `/ops/traces/{trace_id}/spans`는 단계 순서대로 저장된 trace span을 반환하며 다른 워크스페이스에서는 조회되지 않는다.
- `/ops/observability/exports`는 trace span, 품질 감사, 최종 추천 요약을 `specpilot.observability.v1` payload로 저장하고 외부 APM exporter가 읽을 outbox를 제공한다.
- `/ops/observability/dispatch`는 queued/failed export를 선택해 OpenTelemetry/LangSmith exporter outbox로 dispatch하고 성공, 실패, 재시도 예정 상태를 기록한다.
- 관리자 콘솔은 Trace 저장소 섹션에서 최근 분석의 span 수와 경고/차단 수를 보여준다.
- 관리자 콘솔은 Observability export outbox 섹션에서 적재된 trace payload, 대상 destination, span 수, 품질 점수, dispatch 메시지, 재시도 상태를 보여준다.
- `/reports/{report_id}/share`는 워크스페이스 소유자가 저장 리포트에 공개 토큰을 발급하게 한다.
- `/public/reports/{share_token}`과 `/r/{share_token}`은 API 키 없이 단일 공개 리포트만 조회하게 한다.
- 공유 해제 시 기존 공개 토큰은 404가 되며 워크스페이스 내부 리포트 목록은 노출되지 않는다.
- `/reports/completion-templates`와 `/reports/completion-recipient-groups`는 완료 리포트 provider별 제목/본문 템플릿, 수신자 그룹, unsubscribe 제외 정책을 워크스페이스별로 저장한다.
- `/reports/completion-preview`는 저장 리포트와 템플릿, 수신자 그룹을 조합해 발송 전 제목, 본문, 마스킹된 수신/제외 대상, 공개 리포트 경로를 렌더링한다.
- `/reports/completion-batches`는 저장된 구매 리포트를 템플릿과 수신자 그룹 기준으로 이메일, 웹훅, 문자 outbox에 batch 발송하고 개별 delivery 성공, 실패, skip, 재시도 예정 상태를 기록한다.
- 각 completion delivery는 provider 삽입용 `tracking_pixel_path`와 `tracking_click_path`를 발급한다. `/t/o/{tracking_token}.png`는 인증 없이 1x1 PNG를 반환하며 open 이벤트를 기록하고, `/t/c/{tracking_token}`은 안전한 상대 경로만 대상으로 click 이벤트를 기록한 뒤 리다이렉트한다.
- `/reports/completion-deliveries/{delivery_id}/engagement`와 `/reports/completion-engagement`는 완료 리포트 delivery의 열람, 클릭, 공유, 회신 이벤트를 워크스페이스별로 기록하고 조회한다.
- `/reports/completion-deliveries/provider-webhooks`는 `COMPLETION_WEBHOOK_SECRET`으로 보호되는 provider webhook을 받아 delivered, bounced, complained, suppressed, deferred, dropped 이벤트를 저장하고 delivery 상태와 운영 지표에 반영한다.
- `/reports/completion-provider-events`는 워크스페이스별 provider webhook 이벤트, provider명, 상태, 메시지를 조회한다.
- 관리자 콘솔은 완료 리포트 배치 섹션에서 템플릿, 수신자 그룹, 최근 batch의 선택 수, 성공/실패 수, 수신 대상 마스킹, delivery 상태, 추적 픽셀/클릭 path, 열람/클릭 수, 반송/신고/수신 제외 수, 최근 engagement와 provider webhook 이벤트를 보여준다.
- `report.purchase_decision`은 점수, 목표가, 예산 초과, 호환성 차단, 출처 검수 신호를 종합해 구매 진행 가능, 가격 대기, 검수 후 구매로 나눈다.
- 웹 결과 화면과 공개 리포트는 구매 판정, 확신도, 결제 전 체크리스트를 추천 결과 앞에 표시한다.
- `report.share_brief`는 공유받은 사람이 바로 검토할 판정, 핵심 근거, 리스크, 질문, 복사용 문구를 반환한다.
- 웹 결과 화면과 공개 리포트는 공유용 검토 브리프를 추천 카드 직후에 표시한다.
- `report.scenario_options`는 예산 절감, 성능 우선, 안전 우선 기준의 대안 후보와 트레이드오프를 반환한다.
- 웹 결과 화면과 공개 리포트는 TOP 3 외에도 대안 시나리오 카드를 표시한다.
- `report.criteria_matches`는 필수 조건과 제외 조건의 후보별 충족률, 경고, 차단 사유를 반환한다.
- 웹 결과 화면과 공개 리포트는 조건 충족 매트릭스를 비교표와 함께 표시한다.
- `report.stress_tests`는 예산 10% 절감, 예산 10% 여유, 조건 강화 시 선택 유지, 변경, 대기 판단을 반환한다.
- 웹 결과 화면과 공개 리포트는 스트레스 테스트 카드를 대안 시나리오 다음에 표시한다.
- `report.deal_windows`는 후보별 현재가, 목표가, 적정가 밴드, 변동 리스크, 대기 이유, 결제 트리거를 반환한다.
- 웹 결과 화면과 공개 리포트는 구매 타이밍 윈도우를 공유 브리프 다음에 표시한다.
- `report.evidence_packs`는 후보별 가격 계산, 리뷰 근거 수, 벤치마크, 호환성, 출처 신뢰, 검수 필요 여부를 반환한다.
- 웹 결과 화면과 공개 리포트는 근거 팩을 스트레스 테스트 다음에 표시한다.
- `report.option_audits`는 후보별 핵심 사양 기대값, 확인 방법, 옵션 불일치 리스크, 구매 차단 항목을 반환한다.
- 웹 결과 화면과 공개 리포트는 옵션/사양 검수표를 근거 팩 다음에 표시한다.
- `report.execution_plan`은 결제 전 실행 단계, 판매자 확인 질문, 공유 검토 문구를 반환한다.
- 웹 결과 화면과 공개 리포트는 구매 실행 패키지를 구매 판정 직후 표시한다.
- `/reports/{report_id}/checkout-review`는 저장 리포트의 선택 후보에 대해 최종 결제 금액, 옵션/사양 검수, 판매자 답변, 리스크 승인 누락을 계산해 결제 가능, 확인 필요, 보류 상태를 저장한다.
- `/reports/{report_id}/checkout-reviews`와 `/checkout-reviews`는 워크스페이스별 결제 전 검수 이력을 반환한다.
- 관리자 콘솔은 결제 전 검수 섹션에서 readiness 점수, 결제 보류 여부, 누락된 리스크 승인, 항목별 검수 상태를 보여준다.
- `/reports/{report_id}/purchase-outcomes`는 저장 리포트의 선택 후보에 대해 실제 구매, 이탈, 지연, 반품/취소 상태와 최종 결제 금액 차이, 만족도, 주문번호 마스킹값, 학습 신호를 저장한다.
- `/reports/{report_id}/purchase-outcomes`와 `/purchase-outcomes`는 워크스페이스별 구매 결과 이력을 반환한다.
- 관리자 콘솔은 구매 결과 섹션에서 상태, 최종 결제 금액, 예상가 대비 차이, 만족도, 학습 신호를 보여준다.
- `/ops/learning-insights`는 구매 결과, 결제 전 검수 차단, 피드백 만족도를 제품별로 묶어 전환율, 반품률, 가격 차이, 전환 금액, 학습 태그, 추천 개선 액션을 반환한다.
- 관리자 콘솔은 학습 인사이트 섹션에서 제품별 전환/반품/가격 신호와 다음 운영 액션을 보여준다.
- `/feedback`은 추천 만족도, 구매 의향, 선택 후보, 개선 요청을 저장한다.
- `/beta/leads`는 베타 신청 리드를 저장하되 이메일 원문은 저장하지 않고 마스킹된 값만 남긴다.
- `/pricing/plans`는 Free 리포트, Premium 구매 코치, Team 구매 보조 요금제와 가격, 대상 사용자, 기능 목록을 반환한다.
- `/billing/subscription-intents`는 Premium/Team 관심 등록, 결제 주기, 팀 규모, 최대 예산, 기능 우선순위, 구매 시점, 예상 MRR을 워크스페이스별로 저장한다.
- `/ops/pricing-dashboard`는 구독 의향 수, 유료 요금제 의향 수, 예상 MRR, 연환산 매출, 평균 예산, 상위 요금제, 결제 연동 전 다음 액션을 반환한다.
- `/beta/readiness`는 분석 실행, 저장/공유 리포트, 공개 조회수, 알림 연결, 피드백, 베타 리드, 품질 차단 사유를 launch readiness 점수와 체크 항목으로 반환한다.
- `/beta/launch-gate`는 readiness, 품질 회귀, 제품별 학습 인사이트, 백로그 SLA, 구매 전환, 리포트 발송/알림 운영 상태를 묶어 go, limited_beta, hold, blocked 판정과 필수 액션을 반환한다.
- `/beta/cohorts`는 구매 시나리오별 베타 테스트군을 만들고 persona, 키워드, 카테고리 기준으로 리드와 피드백을 집계한다.
- `/beta/backlog`는 readiness 경고, 낮은 만족도 피드백, 품질 차단 사유, 제품별 학습 인사이트에서 개선 백로그를 자동 생성한다.
- `/beta/backlog/{backlog_id}`는 자동 백로그에 운영 상태, 담당자, SLA, 완료 요약, 메모를 워크스페이스별로 덧씌운다.
- `/beta/backlog/summary`는 open, 진행, 완료, 지연, 마감임박, blocker 수와 최근 완료 요약을 반환한다.
- `/beta/cohorts/{cohort_id}/report`와 `/beta/cohorts/{cohort_id}/report.md`는 cohort 지표, 다음 액션, 열린 백로그를 JSON/Markdown으로 export한다.
- 운영 지표는 피드백 수, 평균 만족도, 구매 의향률, 실제 구매 결과 수, 구매 전환율, 최종가 평균 차이, 전환 금액, 제품별 학습 인사이트, 베타 리드 수, 공유 리포트 수, 공개 조회수를 함께 노출한다.
- 관리자 콘솔은 최근 사용자 피드백과 베타 리드를 별도 섹션으로 보여주고, 베타 출시 준비도, 출시 게이트, cohort 리포트 export, 개선 백로그 운영 상태와 다음 실행 항목을 노출한다.

정책 기준:

| 항목 | 기준 |
|---|---|
| 조건 진단 | 예산, 목적, 필수 조건, 카테고리별 맥락이 부족하면 분석 전 질문과 보강 조건을 제시한다 |
| 가격 캐시 | 가격 소스는 45-180분 단위로 캐시하고 만료 시 구매 직전 재확인을 요구한다 |
| 리뷰 근거 | 단정 표현 대신 반복 불만, 리스크 신호, 근거 수를 표시한다 |
| 벤치마크 | 목적별 성능 판단에 쓰되 가격 순위보다 우선하지 않는다 |
| 후보 근거 팩 | 가격, 리뷰, 벤치마크, 호환성, 출처 신뢰를 후보별로 묶어 검증 가능하게 한다 |
| 구매 링크 | 공개 리포트의 구매 링크는 내부 redirect로 클릭을 기록하고 제휴/비제휴 여부를 표시한다 |
| 제휴 고지 | 제휴 링크가 붙더라도 추천 순위는 목적, 가격, 호환성, 리뷰 신뢰도로 계산하고 비제휴 대안을 함께 제공한다 |
| 관리자 검수 | 낮은 신뢰도와 리스크 플래그가 있는 근거는 공개 추천 전에 검수한다 |
| 외부 URL 인입 | 내부망/private IP URL을 차단하고 실제 상품 URL/HTML 스냅샷에서 추출한 가격 근거는 검수 큐를 거친다 |
| URL 모니터 | 반복 수집 대상 URL의 refresh 성공/실패와 검수 큐 연결 상태를 워크스페이스별로 저장한다 |
| Due refresh | cadence와 마지막 실행 시각 기준으로 due 모니터만 실행해 불필요한 외부 호출을 줄인다 |
| Provider 정책 | live fetch는 승인된 host, robots/약관 승인, 시간당 rate limit을 통과해야 실행한다 |
| 공유 리포트 | 공개 토큰은 단일 저장 리포트만 열며 소유 워크스페이스만 생성과 해제가 가능하다 |
| 구매 판정 | 중대한 호환성 또는 출처 검수 신호가 있으면 결제 전 검수로 판정한다 |
| 대안 시나리오 | 1순위를 강요하지 않고 예산, 성능, 안전 우선 관점의 합리적 대안을 함께 제공한다 |
| 스트레스 테스트 | 예산과 조건 변화에 취약한 추천은 유지, 변경, 대기 판단을 명확히 노출한다 |
| 구매 타이밍 | 현재가, 목표가, 적정가 밴드, 재고/쿠폰 변동 리스크를 기준으로 즉시 결제와 가격 대기를 구분한다 |
| 구매 결과 | 주문번호 원문은 저장하지 않고 마스킹하며 실제 구매, 이탈, 지연, 반품/취소 신호와 최종가 차이를 운영 지표와 개선 신호로만 사용한다 |
| 학습 인사이트 | 구매 결과, 결제 전 검수 차단, 피드백 만족도를 제품별 개선 액션으로 집계하되 개인 주문 원문이나 민감 연락처는 포함하지 않는다 |
| 알림/관측 dispatch | 실제 외부 채널과 APM 전송 전 큐 이벤트를 저장하고 dispatch, 성공/실패, 재시도 기준을 남기며 개인정보는 마스킹한다 |
| 품질 감사 | 품질 점수와 예상 비용을 저장하고 공개 차단 사유가 있으면 운영자가 먼저 확인한다 |
| 품질 회귀 | 최근/이전 분석 품질, 비용 변화, provider 차단율을 비교해 release health와 다음 액션을 계산한다 |
| Trace 저장 | 각 분석의 LangGraph 단계 span을 별도 저장하고 워크스페이스별로만 조회하게 한다 |
| 베타 피드백 | 연락처와 이메일 원문을 저장하지 않고 추천 만족도와 구매 의향을 전환 지표로 본다 |
| 구독 의향 | 이메일 원문은 저장하지 않고 요금제, 결제 주기, 팀 규모, 예산, 예상 MRR만 집계해 결제 연동 전 수요 판단에 사용한다 |
| 베타 준비도 | 분석 실행, 공유 조회, 알림 연결, 피드백, 리드, 품질 차단 사유를 함께 평가해 공개 확대 여부를 판단한다 |
| 출시 게이트 | readiness, 품질 회귀, 학습 인사이트, 백로그 SLA, 전환/발송 운영 상태에 blocker가 있으면 공개 확대를 중단하고 필수 액션을 먼저 처리한다 |
| 베타 cohort | 구매 시나리오별 리드와 피드백을 묶고 readiness/피드백/품질/학습 인사이트 신호에서 개선 백로그를 만든다 |
| 베타 운영 액션 | cohort 리포트는 JSON/Markdown으로 export하고 백로그 상태, 담당자, SLA, 완료 요약, 메모는 워크스페이스별로 격리한다 |

## 최종 산출물 정의

사용자는 한 번의 요청으로 분석 전 조건 진단을 받고, 누락된 예산, 목적, 필수 조건, 데스크톱/노트북 맥락을 보강한 뒤 후보 5개를 수집하고, 최종 추천 3개와 제외 후보 2개를 비교표로 확인한다. 각 추천에는 실구매가, 핵심 장점, 반복 단점, 호환성 점검, 적합 사용자, 구매 전 확인할 점, 출처 링크가 포함된다. 운영자는 실제 상품 URL 또는 HTML 스냅샷을 인입해 가격과 페이지 근거를 검수 큐에 올리고, 반복 확인이 필요한 URL은 모니터로 등록해 cadence 기준 due refresh와 검수 큐 연결 상태를 추적한다. live fetch는 provider host 승인, robots/약관 검토, 시간당 rate limit을 통과한 경우에만 실행한다. 승인된 근거만 추천 신뢰 장치에 반영한다. 공유용 검토 브리프는 판정, 핵심 근거, 확인할 점, 검토 질문, 복사용 문구를 제공해 공개 리포트를 받은 사람이 긴 리포트 전에 판단하게 한다. 최종 리포트는 후보별 근거 팩으로 가격 계산, 리뷰 근거 수, 벤치마크, 호환성, 출처 신뢰를 한 화면에서 검증하게 한다. 옵션/사양 검수표는 판매 페이지 옵션명, 장바구니 구성표, 최종 결제 화면의 핵심 사양을 같은 기대값으로 대조하게 한다. 추가로 구매 진행 가능, 가격 대기, 검수 후 구매 중 하나의 판정과 확신도, 위험 플래그, 다음 행동을 제공한다. 구매 타이밍 윈도우는 후보별 현재가, 목표가, 적정가 밴드, 재고/쿠폰 변동 리스크, 결제 트리거를 묶어 즉시 결제와 가격 대기 판단을 분리한다. 저장 리포트 기반 구매 링크는 후보별 제휴/비제휴 판매처, 가격, 쿠폰, 제휴 고지, 정책 경고를 공개 리포트에 노출하고 `/buy/{link_id}` redirect로 클릭 지표를 남긴다. 제휴 링크가 있으면 같은 후보의 비제휴 대안을 함께 제공해야 하며 추천 순위 계산에는 제휴 여부나 클릭 수를 직접 반영하지 않는다. 저장 리포트 기반 결제 전 검수는 최종 결제 금액, 옵션/사양 검수, 판매자 답변, 리스크 승인 누락을 기록해 결제 가능, 확인 필요, 보류 상태를 남긴다. 구매 결과 추적은 실제 구매, 이탈, 지연, 반품/취소 상태와 최종 결제 금액 차이, 만족도, 주문번호 마스킹값, 학습 신호를 저장해 추천 성과를 닫힌 루프로 검증한다. 학습 인사이트는 구매 결과, 결제 전 검수 차단, 피드백 만족도를 제품별 전환율, 반품률, 가격 신선도, 다음 개선 액션으로 묶어 운영자가 추천 노출과 근거 보강 우선순위를 정하게 한다. 저장된 완료 리포트는 발송 전 템플릿/수신자 그룹 기준으로 렌더링 미리보기를 확인한 뒤 운영 채널 outbox에 전달되고 provider 삽입용 추적 픽셀/클릭 리다이렉트 path를 발급한다. unsubscribe 대상은 skip delivery로 남으며 성공, 실패, 재시도 예정, 열람, 클릭, 반송, 신고, 수신 제외 상태가 기록된다. 목표가 도달 알림은 큐 이벤트와 채널별 dispatch 시도를 남겨 운영자가 발송 성공, 실패, 재시도 기준을 추적하게 한다. 또한 예산 절감, 성능 우선, 안전 우선 시나리오와 조건 충족 매트릭스, 예산/조건 스트레스 테스트, 구매 실행 패키지를 함께 제시해 사용자가 자기 우선순위와 입력 조건에 맞는 선택을 실제 구매 행동으로 전환하게 한다.

공개 베타 리포트와 운영 콘솔에는 추가로 출처 신뢰 등급, 캐시 TTL, 제휴 고지, 구매 링크 클릭 지표, 제휴/비제휴 대안 정책 경고, 검수 필요 여부, 품질 감사 점수, 예상 비용, 품질 회귀 상태, provider 차단율, trace span, observability export outbox dispatch/retry, 외부 연동 provider의 mock/configured/verified/blocker 준비도, 완료 리포트 템플릿/수신자 그룹/batch delivery/engagement/provider webhook 상태, 공유 링크 조회수, 추천 만족도, 구매 의향, 실제 구매 결과, 구매 전환율, 최종가 차이, 제품별 학습 인사이트, 베타 신청 전환 지표, Free/Premium/Team 구독 의향, 예상 MRR, 연환산 매출, launch readiness 점수, readiness/품질 회귀/학습 인사이트/백로그 SLA/전환/발송/외부 연동 운영 상태를 묶은 출시 게이트 go/no-go, 시나리오별 cohort 집계, JSON/Markdown cohort 리포트 export, readiness/피드백/품질/학습 인사이트 기반 자동 개선 백로그, 백로그 담당자/상태/SLA/완료 요약/메모가 포함된다. 이는 최저가 추천보다 구매 실패 가능성을 줄이는 제품 포지션을 강화한다.
